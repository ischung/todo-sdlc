## 1. 프로젝트 개요

### 1.1 Core Goal
대학생/취준생이 흩어진 할 일을 놓치지 않고, 월간 달력 한 화면에서 날짜별로 빠르게 기록·확인할 수 있게 한다.

### 1.2 배경 및 목적
- **대상 사용자**: 한 학기에 강의·과제·스터디·알바·취업 준비 일정을 병행하는 대학생/취준생
- **사용자 Pain Point**: 카톡 공지, 강의 안내, 메모장에 할 일이 흩어져 있어 과제·자소서·면접 마감을 자주 놓친다
- **기대 효과**: 달력 칸의 개수 배지만 봐도 "내일 할 일 3개 있었지" 하고 즉시 떠올라 마감을 놓치지 않는다

### 1.3 범위 (Scope)

**In-Scope (v1 포함)**
- F1. 월간 달력 화면 (이번 달 보기, 이전/다음 달 이동)
- F2. 날짜 클릭 → 할 일 추가 (한 줄 입력)
- F3. 할 일 완료 체크 (토글)
- F4. 할 일 삭제
- F5. 날짜 칸에 할 일 개수 배지 표시
- F6. 새로고침해도 데이터 유지 (영속 저장)
- F8. 오늘 날짜 강조 표시

**Out-of-Scope (v1 제외)**
- 할 일 수정 (이유: 삭제 후 재입력으로 우회 가능, v1 복잡도 ↓)
- 반복 일정 (이유: 캘린더 앱의 큰 복잡도, Core Goal 기여 미미)
- 알림/푸시 리마인더 (이유: 브라우저·모바일 권한 인프라 부담)
- 카테고리·태그·색상 라벨 (이유: 망각 방지 핵심에 직접 기여하지 않음)
- 다중 사용자/로그인/계정 동기화 (이유: 개인용 단일 사용자 가정)
- 여러 기기 간 동기화 (이유: 로컬 영속 저장 기준)
- 마감 시각 (시:분 단위) (이유: 일자 단위 표시로 v1 충분)
- 검색/필터 (이유: v1 데이터량에서 불필요)
- 내보내기 (CSV·이미지) (이유: v1 불필요)
- 드래그로 다른 날짜 이동 (이유: v1 복잡도 ↑)

## 2. 대상 사용자

**Primary Persona**: 이지원 (22세, 대학교 4학년 졸업 학기 / 취준 병행)

> 한 학기에 4과목 + 자소서·면접 일정 + 스터디 + 알바를 병행한다.
> 할 일이 카톡 공지·강의 안내·종이 다이어리 사이에 흩어져 있어 과제 마감, 자소서 마감, 면접 일정을 자주 놓친다.
> 무거운 캘린더 앱(구글 캘린더, 노션) 대신 '한 달이 한눈에 보이는 가벼운 도구'를 원한다.

**핵심 니즈**
- 기능적: 날짜별로 할 일을 빠르게 적고/지우고 싶다, 한 달 분포를 한눈에 보고 싶다
- 감성적: 복잡한 설정 없이 즉시 쓸 수 있는 가벼움

## 5. 상세 기능 요건

### 5.1 F1. 월간 달력 화면

**Logic**
1. 앱 진입 시 오늘이 포함된 달의 7×N 그리드를 표시한다.
2. 각 칸에는 날짜 숫자, 해당 날짜의 미완료 할 일 개수 배지(F5), 오늘 날짜 강조(F8)가 함께 표시된다.
3. 헤더의 "이전 달 / 다음 달" 버튼으로 표시 월을 전환할 수 있다. "오늘로 이동" 버튼으로 즉시 현재 월로 복귀한다.

**Validation**
- 표시 가능 범위: 제한 없음 (사용자가 임의 과거·미래 월로 이동 가능)

**Edge Case**
- 데이터 없음: 어떤 날짜에도 할 일이 없으면 개수 배지를 표시하지 않는다 (빈 칸 자체가 정상 상태)
- 오류 발생: 저장소 읽기 실패 시 "할 일을 불러오지 못했어요. 새로고침 해주세요." 안내

### 5.2 F2. 날짜 클릭 → 할 일 추가

**Logic**
1. 달력에서 날짜 칸을 클릭하면 해당 날짜의 상세 패널(또는 모달)이 열린다.
2. 패널 내 입력창에 할 일을 한 줄로 입력하고 Enter 또는 "추가" 버튼을 누르면 목록에 즉시 추가된다.
3. 추가 직후 입력창은 비워지고 포커스가 유지되어 연속 입력이 가능하다.

**Validation**
- 제목: 1자 이상, 최대 100자 / 공백만 입력 시 추가 불가
- 빈 입력으로 Enter 시 아무 동작도 하지 않는다 (시각적 흔들림으로 가벼운 피드백)

**Edge Case**
- 데이터 없음: 해당 날짜에 할 일이 없으면 "아직 할 일이 없어요. 첫 번째 항목을 추가해보세요!"
- 오류 발생: 저장 실패 시 "저장하지 못했어요. 다시 시도해주세요." + 재시도 버튼

### 5.3 F3. 할 일 완료 체크 (토글)

**Logic**
1. 상세 패널의 각 할 일 항목 좌측 체크박스를 클릭하면 완료 상태로 토글된다.
2. 완료된 항목은 취소선 + 회색 톤으로 표시한다.
3. 다시 클릭하면 미완료로 되돌아간다.

**Validation**
- 별도 입력값 없음

**Edge Case**
- 데이터 없음: 해당 사항 없음
- 오류 발생: 상태 저장 실패 시 토글을 즉시 원복하고 "변경을 저장하지 못했어요." 토스트

### 5.4 F4. 할 일 삭제

**Logic**
1. 각 할 일 항목 우측 "삭제" 아이콘을 클릭하면 즉시 목록에서 제거된다.
2. 삭제 직후 5초 동안 "되돌리기" 버튼을 토스트로 노출한다(실수 방지).
3. 5초 경과 또는 다른 동작 발생 시 영속 삭제로 확정된다.

**Validation**
- 별도 입력값 없음

**Edge Case**
- 데이터 없음: 해당 사항 없음
- 오류 발생: 삭제 저장 실패 시 항목을 즉시 원복하고 "삭제하지 못했어요." 토스트

### 5.5 F5. 날짜 칸 개수 배지

**Logic**
1. 각 날짜 칸 우상단에 해당 날짜의 미완료 할 일 개수를 배지로 표시한다.
2. 0개이면 배지를 숨긴다. 9개 초과 시 "9+"로 표시한다.
3. 할 일 추가/완료/삭제 시 배지 숫자가 즉시 갱신된다.

**Validation**
- 별도 입력값 없음

**Edge Case**
- 데이터 없음: 배지 미표시
- 오류 발생: 데이터 로드 실패 시 배지를 일시적으로 숨김

### 5.6 F6. 영속 저장

**Logic**
1. 모든 할 일 추가/체크/삭제 동작은 발생 즉시 로컬 영속 저장소에 반영된다.
2. 앱 진입 시 저장소에서 데이터를 읽어 달력에 반영한다.
3. 새로고침·브라우저 재실행 후에도 데이터가 그대로 보인다.

**Validation**
- 저장 용량 한계 도달 시 사용자에게 안내

**Edge Case**
- 데이터 없음: 최초 진입 시 빈 달력 + "첫 번째 할 일을 추가해보세요!" 빈 상태 안내
- 오류 발생: 저장 실패 시 토스트로 알리고 재시도 유도

### 5.7 F8. 오늘 날짜 강조

**Logic**
1. 현재 표시 월에 '오늘'이 포함되어 있으면 해당 칸의 날짜 숫자와 테두리를 강조 색으로 표시한다.
2. 자정이 지나면 다음날 진입 시 강조 위치가 자동 갱신된다.

**Validation**
- 별도 입력값 없음

**Edge Case**
- 데이터 없음: 해당 사항 없음
- 오류 발생: 시간 정보 획득 실패 시 강조를 생략한다 (앱 동작에는 영향 없음)

### 5.8 인터랙션 요약

| 사용자 행동 | 시스템 반응 |
|-------------|-------------|
| 달력에서 날짜 클릭 | 해당 날짜 상세 패널 열림 |
| 입력창에 텍스트 + Enter | 할 일 즉시 추가, 입력창 비워짐, 배지 +1 |
| 할 일 좌측 체크박스 클릭 | 완료 토글, 취소선·회색 표시, 미완료 배지 카운트 갱신 |
| 할 일 우측 삭제 클릭 | 즉시 목록에서 제거 + 5초 되돌리기 토스트 |
| "이전/다음 달" 클릭 | 표시 월 전환, 칸·배지 즉시 갱신 |
| "오늘로 이동" 클릭 | 현재 월로 복귀 + 오늘 칸 강조 |
| 새로고침 | 저장된 데이터 그대로 복원 |

## 6. UI/UX 요건

- **레이아웃**: 단순형 — 메인 화면은 월간 달력 한 화면. 날짜 클릭 시 상세 패널(또는 모달) 1단계 노출
- **로딩 상태**: 데이터 로드 중에는 가벼운 스피너(회전 아이콘) 표시
- **오류 메시지 톤**: 친근하고 가볍게 ("앗, 저장하지 못했어요. 다시 시도해주세요." 같은 형태)
- **빈 상태 안내**: 항목이 없는 날짜 패널에 "아직 할 일이 없어요. 첫 번째 항목을 추가해보세요!" 노출
- **모바일 대응**: 좁은 화면에서도 7열 달력 그리드를 유지하며, 상세 패널은 하단 시트 형태로 전환 가능

## 7. 기술적 제약

- **지원 환경**:
  - 웹 브라우저 (최신 2개 메이저 버전: Chrome, Safari, Edge)
  - 데스크탑 우선 + 모바일 반응형 (별도 네이티브 앱 없음)
- **성능 목표**:
  - 첫 화면(월간 달력) 로딩 2초 이하
  - 할 일 추가/완료/삭제 반응 200ms 이하
- **데이터 보존**: 사용자의 브라우저 환경에 한해 영속 보존 (구체 저장 방식은 TechSpec에서 결정)
- **접근성**: 키보드만으로도 추가/체크/삭제 가능
